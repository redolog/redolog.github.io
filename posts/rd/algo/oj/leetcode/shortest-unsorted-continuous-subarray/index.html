题目#

581. Shortest Unsorted Continuous Subarray#

https://leetcode.cn/problems/shortest-unsorted-continuous-subarray/

Given an integer array nums, you need to find a continuous subarray that, if you only sort this subarray in ascending order, will result in the entire array being sorted in ascending order.

You are required to return the length of the shortest subarray that needs to be sorted.

解释下题目：

  • 给定一个数组nums；
  • 要求找到一个连续子数组，这个子数组正序排序后，整个数组正向有序；
  • 需要返回此子数组的长度；

读完题，我们可以知道，这个数组除了一些特殊情况（单调递增、单调递减），整体分为两部分：

  • 左右两侧的正序区间；
  • 中间的乱序区间；

为了方便描述，我们假设中间的乱序区间为：(l, r)，那么两侧的正序区间则分别为：[0, l]、[r, n-1]。

解法1：整体排序后匹配#

思路#

既然最后的效果是想要整体有序的数组，那么我们可以对数组排序，然后分别从左侧、右侧找两个数组不一样的元素，这两个不一样的元素，就是l、r。

这种思路简单直接并且有效！

// 此解法的中间区间为 `[l,r]`
func findUnsortedSubarray(nums []int) int {
	n := len(nums)
	sortedNums := make([]int, n)
	for i := 0; i < n; i++ {
		sortedNums[i] = nums[i]
	}
	sort.Ints(sortedNums)

	l, r := 0, n-1
	for l <= r && nums[l] == sortedNums[l] {
		l++
	}
	for r >= l && nums[r] == sortedNums[r] {
		r--
	}
	return r - l + 1
}

复杂度分析#

时间#

排序占用主要时间，时间复杂度为O(n*logn)。

空间#

复制了一份新的数组，空间复杂度O(n)。

解法2：拆分数组几个部分#

思路#

观察下代表性的用例数据：

最后一个用例为题目重点关注的代表性用例。

如最开始提到的，给定的数组可以拆解为：

  • 中间的乱序区间：(l, r)；
  • 两侧的正序区间则分别为：[0, l]、[r, n-1]；

进一步推导：

  • 我们将三个区间分别叫做：L、M、R；
  • 想要满足题目条件，需同时满足：
    • max(L)<min(M)
    • min(R)>max(M)

我们可以先初始化一个大概的(l, r)。

从左往右找，区间一直升序，找到第一个nums[i]>nums[i+1]的位置，此时i==l。

同理，从右往左找，区间一直升序，找到第一个nums[i-1]>nums[i]的位置，此时i==r。

此时的l r均破坏了左右区间的单调升序性质。

但是此时的[l, r]中，有可能存在元素nums[i]<nums[l-1]，由于L区间的单调升序性质，nums[l-1]为左区间最大值。此时，我们只能向左扩大[l, r]，直到nums[i]>nums[l-1]。

同理，右区间也可以这样向右扩张。目的都是为了保证上面我们推导出的性质：

  • max(L)<min(M)
  • min(R)>max(M)

推到这里，我尝试写了一版代码，我直接找出了中间区间的最值，实际上，当向左右扩张区间时，中间区间的最值是在变化的，所以中间区间的极值需要实时更新。

func findUnsortedSubarray(nums []int) int {
	n := len(nums)
	// l为左区间包含边界，r为右区间包含边界
	l, r := 0, n-1
	// 先找出l初始值
	for l+1 <= r && nums[l+1] >= nums[l] {
		l++
	}
	// 再找出r初始值
	for r-1 >= l && nums[r] >= nums[r-1] {
		r--
	}
	// 取左右区间边界作为中间区间极值初始值
	midMin, midMax := nums[l], nums[r]
	// 分别向左向右扩张
    // 由于l、r在循环中会扩张变化，因此开启循环前暂存此范围变量
	il, ir := l, r
	for i := il; i <= ir; i++ {
		if nums[i] < midMin {
			// 中间区间出现了更小的元素，左区间需扩张到所有元素都小于该更小元素
			for l >= 0 && nums[l] > nums[i] {
				l--
			}
			if l < 0 {
				midMin = math.MinInt
			} else {
				midMin = nums[i]
			}
		}
		if nums[i] > midMax {
			// 中间区间出现了更大的元素，右区间需扩张到所有元素都大于该更大元素
			for r < n && nums[r] < nums[i] {
				r++
			}
			if r >= n {
				midMax = math.MaxInt
			} else {
				midMax = nums[i]
			}
		}
	}
	if r == l {
		return 0
	}
	return (r - 1) - (l + 1) + 1
}

上面的写法思路上关键在于：

  • 数组分区；
  • 定义每个分区之间的规律；

而具体到编码层面，有些细节则容易让我们陷入debug。这些细节值得反复斟酌。

比如遍历中间区间时，最开始我没有固定l/r，导致了数组访问越界。

再次分析、观察上述过程与代码，会发现：

  • 左侧升序区间的边界l一定是从右往左开始找；
  • 并且找的过程中需要持续检查左区间、中间区间的性质；
  • 右区间同理；

// 此写法 [l,r] 为中间乱序区间
func findUnsortedSubarray(nums []int) int {
	n := len(nums)
    // 中间区间的极值
	min, max := nums[n-1], nums[0]
	// 先找左边界 l
	l := n - 1
	for i := n - 1; i >= 0; i-- {
		if nums[i] <= min {
			min = nums[i]
		} else {
			l = i
		}
	}

	// 再找右边界 r
	r := 0
	for i := 0; i < n; i++ {
		if nums[i] >= max {
			max = nums[i]
		} else {
			r = i
		}
	}
	if l >= r {
		return 0
	}
	return r - l + 1
}

时间#

空间#

仅使用了常数数量级指针空间，空间复杂度O(1)。
