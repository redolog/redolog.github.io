LeetCode 138. 复制带随机指针的链表 题解，理解链上加链的思路。

题目#

https://leetcode.cn/problems/copy-list-with-random-pointer/

给你一个长度为 n 的链表，每个节点包含一个额外增加的随机指针 random ，该指针可以指向链表中的任何节点或空节点。

构造这个链表的 深拷贝。

题解#

粗暴解#

最先想到的是粗暴解，先构建一个只包含next指针的常规链表，同时用数组存储新创建的节点，这里是为了记录下标->节点的关系。

新的一轮循环中，粗暴计算random对应节点的下标。

public RandomListNode copyRandomList(RandomListNode head) {
    if (head == null) {
        return null;
    }

    RandomListNode dummy = new RandomListNode(-1);
    RandomListNode pre = dummy;
    RandomListNode curr = head;

    List<RandomListNode> nodeList = new ArrayList<>();

    while (curr != null) {
        RandomListNode node = new RandomListNode(curr.val);
        nodeList.add(node);

        pre = pre.next = node;
        curr = curr.next;
    }
    pre.next = null;

    curr = head;
    RandomListNode newCurr = dummy.next;
    while (curr != null) {
        RandomListNode random = curr.random;
        // 计算random位置
        if (random == null) {
            newCurr.random = null;
        } else {
            int randomLeftStep = 0;
            while (random != null) {
                ++randomLeftStep;
                random = random.next;
            }
            newCurr.random = nodeList.get(nodeList.size() - randomLeftStep);
        }

        newCurr = newCurr.next;
        curr = curr.next;
    }

    return dummy.next;
}

使用map直接获取random节点#

上个方案中，29-32行计算random位置的代码可以考虑优化。

使用基于数组下标的访问，我们必须算出下标。

回顾一下我们二轮循环的过程：

  • 遍历链表，链由next串联
  • 原链表、新链表可同步进行，同步next步长即可

循环中只能拿到原链表当前节点、新链表当前节点、链next指针、random指针，同时注意，原链表的random指向了原链中的某个节点（可null），如果我们构建一个层索引，即Map<OldNode,NewNode>结构，通过当前节点即可拿到random原节点、random新节点。

也就是可以通过映射索引，构建新链表的random链。

public RandomListNode copyRandomListWithMap(RandomListNode head) {
    if (head == null) {
        return null;
    }

    RandomListNode dummy = new RandomListNode(-1);
    RandomListNode pre = dummy;
    RandomListNode curr = head;

    Map<RandomListNode, RandomListNode> old2New = new HashMap<>();

    while (curr != null) {
        RandomListNode node = new RandomListNode(curr.val);
        old2New.put(curr, node);

        pre = pre.next = node;
        curr = curr.next;
    }
    pre.next = null;

    curr = head;
    while (curr != null) {
        old2New.get(curr).random = old2New.get(curr.random);

        curr = curr.next;
    }
    return dummy.next;
}

这个实现比方案1的计算下标有效率上的提升。时间复杂度最差为O(n)，但是依赖了额外的map，空间复杂度为O(n)。

链上加链#

到现在我们目标更加明确：优化方案2中的空间复杂度。

一般优化空间的思路是：复用原始结构。

我们可以复用原链表结构，构建新的next链时，将新节点暂存在原链上，即将新节点插入到原节点node与node.next之间。

这种操作的妙处：

  1. 保留了原链所有信息（节点、next关系可通过next.next获得、random信息）；
  2. 同时也保留了新链上所需的random信息，原链节点node.next为新节点，node.random.next即为新节点对应random指向；

public RandomListNode copyRandomListWithShadow(RandomListNode head) {
    if (head == null) {
        return null;
    }
    // 先遍历，创建链表影子节点，设置next
    RandomListNode curr = head;
    while (curr != null) {
        RandomListNode nextTmp = curr.next;
        RandomListNode currNew = new RandomListNode(curr.val);
        curr.next = currNew;
        currNew.next = nextTmp;
        curr = nextTmp;
    }
    // 当前节点.next此时为新创建的影子节点，而.random.next即为新创建的random新节点，因此可以再遍历一次，设置影子节点的random
    // 设置random、分离新旧链表指针需分开
    curr = head;
    while (curr != null) {
        curr.next.random = curr.random == null ? null : curr.random.next;

        curr = curr.next.next;
    }
    RandomListNode dummy = new RandomListNode(-1);
    curr = head;
    while (curr != null) {
        RandomListNode shadow = curr.next;
        if (dummy.next == null) {
            dummy.next = shadow;
        }
        RandomListNode originalNext = shadow.next;
        // 影子节点next为null
        if (originalNext != null) {
            shadow.next = originalNext.next;
        }
        curr.next = originalNext;
        curr = curr.next;
    }

    return dummy.next;
}

由于没有额外内存申请，空间复杂度直降为O(1)。

小结#

  • 使用Map可增加一层映射关系；
  • 链上加链的核心：复用原始数据结构，链表结构可在原始基础上增加节点插入；
