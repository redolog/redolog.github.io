题目#

3. 无重复字符的最长子串#

https://leetcode.cn/problems/longest-substring-without-repeating-characters/

给定一个字符串 s ，请你找出其中不含有重复字符的 最长子串 的长度。

DP#

思路#

看到是要求最长子串，我首先往DP的思路上靠了靠，我们设 dp[i]表示s[i]结尾的无重复字符的最长子串长度，看看对应状态变化：

abcabcbb用例： abcabcbb用例状态说明无需对比当前字符与prev字符的关系。

abba用例： abba用例说明：前序已经有别的重合项情况下，不能取当前字符距离，需由上个dp状态叠加。

public int lengthOfLongestSubstring(String s) {

    if (s == null || s.length() == 0) {
        return 0;
    }

    int n = s.length();
    int[] dp = new int[n];
    dp[0] = 1;
    // 记录字符上一个下标
    Map<Character, Integer> c2Idx = new HashMap<>();
    c2Idx.put(s.charAt(0), 0);

    int max = 1;
    for (int i = 1; i < n; i++) {
        char currC = s.charAt(i);
        if (c2Idx.containsKey(currC)) {
            dp[i] = Math.min(dp[i - 1] + 1, i - c2Idx.get(currC));
        } else {
            dp[i] = dp[i - 1] + 1;
        }

        c2Idx.put(currC, i);
        max = Math.max(max, dp[i]);
    }
    return max;
}

复杂度分析#

时间#

设 n == s.length()

整个过程对入参字符串完整遍历一次，时间复杂度O(n)。

空间#

dp[]、map为额外空间，空间复杂度O(n)。

双指针–滑动窗口#

思路#

看了官解，发现不需要推导动态规划的递推状态，仅需双指针，维护一个无重复子串的窗口即可，也叫「滑动窗口」。

abcabcbb用例：

  • 慢指针代表每个窗口起点，我们依次从[0,n-1]滑动。
  • 快指针代表每个窗口终点：
    • 每个窗口进入时，先删掉上个窗口的起始字符，每个终点必定在上个终点之后（仅需求最大窗口）；
    • 每个窗口内我们维护值状态，快指针一直滑到窗口内无重复的节点；

public int lengthOfLongestSubstring(String s) {
    int ans = 0;
    int n = s.length();
    Set<Character> set = new HashSet<>();
    int r = 0;
    for (int l = 0; l < n; l++) {
        // 除了第一轮，后续每次先删除l对应字符
        if (l != 0) {
            set.remove(s.charAt(l - 1));
        }
        // r右移
        while (r < n && !set.contains(s.charAt(r))) {
            set.add(s.charAt(r));
            r++;
        }
        ans = Math.max(ans, r - l);
    }

    return ans;
}

复杂度分析#

时间#

设 n == s.length()

快慢指针均对字符串遍历完成一次，时间复杂度O(n)。

空间#

set为记录窗口状态的额外空间，空间复杂度O(n)。

双指针–滑动窗口2#

思路#

官解并非最优解。理论上大O复杂度无法更优，但实际运行中，有一个部分剪枝的改善方法。

同样基于滑动窗口思想，我们使用map记录每个字符出现的前一个位置。

以用例abcabcbb为例，我们可以观察：

  • abc bca cab abc最长子串的出现，都伴随着出现了r快指针碰到了重复字符，同时，l慢指针均滑动到了重复字符的下一个位置，即l==map.get(currC)+1；

以用例abba为例：

  • ba最长子串的出现，使用上方的等式计算l位置，会发现有问题，由于a的上次出现到这次出现中间，b已经有过重复，l慢指针已经挪动到了下标2的位置，因此我们需要保证l不后退；

综上，我们使用一个map记录每个字符出现的前一个位置，同时保证l指针的递增性，计算快慢指针之间距离即可。在部分用例如abcdefghaf中，此方法慢指针可以跳过不必要遍历的字符，直接来到当前重复字符的前序位置之后，提升了运行效率。

public int lengthOfLongestSubstring2(String s) {
    int ans = 0;
    int n = s.length();
    // 记录字符最新位置
    Map<Character, Integer> c2Idx = new HashMap<>();
    int l = 0;
    for (int r = 0; r < n; r++) {
        char currC = s.charAt(r);
        if (c2Idx.containsKey(currC)) {
            // 碰到了重复字符，左指针可跳跃到重复字符的上一个位置+1
            // 如果l之前的值比当前重复字符+1更小，说明之前已经碰到过别的重复字符，需要取更右侧的值，跳过前序已经判过重的位置
            l = Math.max(c2Idx.get(currC) + 1, l);
        }
        ans = Math.max(ans, r - l + 1);
        c2Idx.put(currC, r);
    }
    return ans;
}
