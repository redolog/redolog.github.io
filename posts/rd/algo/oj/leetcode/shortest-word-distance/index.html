题目#

243. 最短单词距离#

https://leetcode.cn/problems/shortest-word-distance/

给定一个字符串数组 wordDict 和两个已经存在于该数组中的不同的字符串 word1 和 word2 。返回列表中这两个单词之间的最短距离。

模拟#

思路#

这道题首先一个粗暴的想法是找到两个单词所在位置，因为位置可能是多个，在多个基础上，一一匹配。明显这是一个N^2复杂度的思路。

尝试能否在一次遍历中找到题目所求最短距离？

画个图看看情况分布：

发现，每次我们更新最新的word1 word2位置，同时计算得到最小的距离，在一次遍历内即可完成。

后续位置有更新时，逻辑也是完备的。

public int shortestDistance(String[] wordsDict, String word1, String word2) {
    int ans = wordsDict.length;
    // word1、word2对应下标位置
    int idx1 = -1, idx2 = -1;
    for (int i = 0; i < wordsDict.length; i++) {
        // 出现word1、word2时，更新最新位置
        if (wordsDict[i].equals(word1)) {
            idx1 = i;
        } else if (wordsDict[i].equals(word2)) {
            idx2 = i;
        }

        if (idx1 >= 0 && idx2 >= 0) {
            ans = Math.min(ans, Math.abs(idx1 - idx2));
        }
    }
    return ans;
}

复杂度分析#

时间#

n表示数组长度

a表示word1字符串长度 a2表示word1出现次数

b表示word2字符串长度 b2表示word2出现次数

整个过程完整遍历一次数组，O(n)时间复杂度。

每次过程匹配单词，复杂度O(max(a,b))。

当word1 word2位置第一次找到后，开始计算最短距离，一次复杂度O(1)，大概进行a2+b2次。

整体时间复杂度O(n * max(a,b))。

空间#

无额外容器占用，空间复杂度大致为O(1)。
